# SeaweedFS BOSH Release

> **⚠️ DEV/TEST ONLY**: This release is for **development and test environments only**. **NOT for production.**
>
> **Single VM = Single Point of Failure. High availability is OUT OF SCOPE by design.**
>
> For production, use **AWS S3**, **Google Cloud Storage**, or **Azure Blob Storage**.

A BOSH release providing [SeaweedFS](https://github.com/seaweedfs/seaweedfs) as an S3-compatible blobstore for Cloud Foundry environments.

## Overview

This release packages SeaweedFS to provide a lightweight, S3-compatible object storage solution for Cloud Foundry **development and test/dev environments**. It is designed as a proof-of-concept to evaluate SeaweedFS as an alternative to singleton-blobstore in local development scenarios.

### Target Use Cases

✅ **Local development** - BOSH Lite, local CF deployments
✅ **Test environments** - CI/CD pipelines, integration testing
✅ **Dev environments** - Development team shared environments
✅ **Evaluation** - Proof of concept, performance testing

❌ **NOT for production** - No high availability, single point of failure
❌ **NOT for staging** - Use external S3 or production-grade solutions
❌ **NOT for production CF** - Use AWS S3, GCS, Azure Blob, or commercial alternatives

### Features

- **S3-Compatible API**: Native S3 protocol support for Cloud Foundry blobstore integration
- **Single-VM Deployment**: Simple architecture perfect for dev/test - one VM, one persistent disk
- **No HA Complexity**: Deliberately simple - HA is out of scope for dev/test use case
- **BBR Integration**: Built-in backup and restore support
- **TLS & mTLS Support**: Secure communication matching CF standards
- **BPM Process Management**: Reliable process supervision following BOSH best practices
- **Auto-generated Credentials**: Zero manual credential management via BOSH variables
- **Automatic Updates**: Renovate integration for dependency version tracking

## Quick Start

```bash
# Upload releases
bosh upload-release https://bosh.io/d/github.com/cloudfoundry/bpm-release?v=1.2.15
bosh upload-release seaweedfs-release.tgz

# Deploy standalone
bosh -d seaweedfs deploy examples/seaweedfs-standalone.yml \
  -v seaweedfs_domain_name=blobstore.system.domain

# Or integrate with cf-deployment
bosh -d cf deploy cf-deployment.yml \
  -o examples/operations/use-seaweedfs-blobstore.yml
```

See [docs/deployment.md](docs/deployment.md) for detailed instructions.

## Architecture

Single-VM deployment with four processes (all managed by BPM):

- **Master Server** (9333): Metadata management
- **Volume Server** (8080): Data storage on persistent disk
- **Filer** (8888): File system layer with LevelDB
- **S3 Gateway** (8333): S3-compatible API endpoint

All data stored on persistent disk at `/var/vcap/store/seaweedfs/`.

## Configuration

### Auto-Generated Credentials

All credentials are auto-generated by BOSH:

```yaml
seaweedfs:
  s3:
    access_key: ((seaweedfs_access_key))      # Auto-generated
    secret_key: ((seaweedfs_secret_key))      # Auto-generated
  tls:
    cert: ((seaweedfs_tls.certificate))       # Auto-generated
    private_key: ((seaweedfs_tls.private_key)) # Auto-generated
```

### mTLS for CF Integration

Enable mutual TLS to match CF security standards:

```yaml
seaweedfs:
  mutual_tls:
    enabled: true
    ca_cert: ((service_cf_internal_ca.certificate))
    verify_client: "require"
```

See [docs/security.md](docs/security.md) for details.

## Integration with Cloud Foundry

Cloud Controller is configured to use S3 API with mTLS:

```yaml
cc:
  buildpacks:
    blobstore_type: s3
    fog_connection:
      provider: AWS
      endpoint: https://blobstore.system.domain:8333
      aws_access_key_id: ((seaweedfs_access_key))
      aws_secret_access_key: ((seaweedfs_secret_key))
      s3_signature_version: "4"
      region: local
```

Use `examples/operations/use-seaweedfs-blobstore.yml` for automatic configuration.

## Backup and Restore

BBR (BOSH Backup and Restore) is included:

```bash
# Backup
bbr deployment --deployment seaweedfs backup

# Restore
bbr deployment --deployment seaweedfs restore \
  --artifact-path seaweedfs_TIMESTAMP/
```

See [docs/deployment.md](docs/deployment.md) for detailed BBR usage.

## Troubleshooting

### S3 API Returns 403 Forbidden
- Verify credentials: `bosh -d seaweedfs manifest | grep seaweedfs_access_key -A 2`
- Check TLS cert is valid
- Ensure Cloud Controller has network access to port 8333

### Processes Not Starting
```bash
bosh -d seaweedfs ssh seaweedfs/0
tail -f /var/vcap/sys/log/seaweedfs/*.stderr.log
```

### mTLS Issues
Verify CA matches between SeaweedFS and CF:
```bash
bosh -d seaweedfs manifest | grep service_cf_internal_ca
bosh -d cf manifest | grep service_cf_internal_ca
```

See [docs/troubleshooting.md](docs/troubleshooting.md) for more details.
